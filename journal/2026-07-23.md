## Vite — triage of #23007 (bundledDev + custom input → 404 at `/`)

Reproduced with a minimal config (no Vue plugin needed) and built the full behavior matrix: classic dev ignores `build.rolldownOptions.input` and serves `index.html` at `/`; bundled dev derives its entries from the build input and loses the root mapping even when `index.html` exists on disk. Root cause is two-layered: `memoryFiles` is keyed by raw bundle `fileName` (only `index-export.html` exists), and `checkFileExists` in `htmlFallbackMiddleware` short-circuits the on-disk fallback via `??` when `memoryFiles` is present.

Since the fix direction is a design call on an experimental feature (dev-entry parity vs fallback mapping), posted the analysis with both directions and offered a PR with regression test: [comment](https://github.com/vitejs/vite/issues/23007#issuecomment-5047160532).
