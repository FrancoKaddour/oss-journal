## Storybook — new fix: coverage crash on large dev-UI runs (#35508)

Root-caused the reported `Something removed the coverage directory` crash: the dev-UI coverage flow runs Vitest in **watch mode** with `cleanOnRerun: true` and a shared `reportsDirectory` in the Storybook cache — any rerun triggered mid-run lets Vitest wipe the directory while workers still write `.tmp` files. Explains why the CLI path doesn't crash (no watch mode) and why only large suites do (wider race window).

Fix: `cleanOnRerun: false`, with explicit cleaning moved to the one point the manager controls — after `cancelCurrentRun()` has drained the previous run. Fresh instances still clean via `clean: true`. Two unit tests added (config contract + safe-point cleaning). Draft PR: [#35544](https://github.com/storybookjs/storybook/pull/35544).

Transparency: the package's unit suite hangs on my Windows environment on a clean checkout (pre-existing), so CI validates the new tests — declared in the PR.
