## Zag #3203 — dialog backdrop click-through (debut)

Root-caused a modal-dialog bug: `disablePointerEventsOutside` sets `body { pointer-events: none }` and relies on inheritance, but a background descendant with an explicit `pointer-events: auto` overrides the inherited `none` and stays clickable under the backdrop (CSS spec behavior). Confirmed the exact line and the layer-stack architecture.

The robust fix is architectural — migrate to the native `inert` attribute on the top-blocking layer's siblings (Radix's approach), which CSS can't override. Since that's a design choice the maintainer should weigh in on, entered the repo with an analysis + proposed direction + offer to implement with an e2e test, rather than a cold architectural PR. First contribution to the Ark/Zag ecosystem. [Comment](https://github.com/chakra-ui/zag/issues/3203#issuecomment-5070431222).
