## Storybook — issue #35405 investigation (JSDoc code fences)

Started from a terse issue by the Storybook maintainer: language-tagged code fences (` ```tsx `) in JSDoc `@example` blocks don't render as code blocks. No repro, no diagnosis.

Probed every layer of the pipeline empirically with the exact locked versions instead of theorizing:

- `comment-parser` — treats tagged and untagged fences identically; `spacing: 'compact'` collapses multi-line tag values onto one line
- `react-docgen` — keeps descriptions verbatim, not involved
- `react-docgen-typescript` — splits examples into `tags` with newlines intact, not involved
- `markdown-to-jsx@7.7.17` (the locked version) — renders both fence styles correctly, not involved

Culprit: `extractJSDocInfo` in the React component manifest parses tags with `spacing: 'compact'`, so a fenced `@example` collapses to one line — untagged fences degrade to an inline code span, language-tagged ones leak the language into the visible text and never render a block. Exactly the asymmetry the issue reported.

Fix: tag values containing a fence fall back to the `preserve` parse. Red-green verified, componentManifest suite 278/278 green. PR: [storybookjs/storybook#35527](https://github.com/storybookjs/storybook/pull/35527) (draft).
