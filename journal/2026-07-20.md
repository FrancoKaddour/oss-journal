## TanStack Start — minimal repro for #7837 (prerender crawler corrupts query strings)

The maintainer had asked for a minimal reproduction and the issue sat in "information needed" for three days. Built one: [tanstack-start-prerender-7837-repro](https://github.com/FrancoKaddour/tanstack-start-prerender-7837-repro).

Findings beyond the original report:

- **Corruption confirmed end-to-end.** The published `@tanstack/start-plugin-core` calls `withTrailingSlash` without `respectQueryAndFragment`, so the crawler requests `/?currency=USD/`; serving that request renders corrupted state (`Currency: USD/`) and emits the next-generation corrupted link (`href="/?currency=USD%2F"`).
- **Why the infinite loop is hard to reproduce minimally (new finding):** instrumenting SSR showed only the *first* crawl reaches the server — the crawler's subsequent requests (including corrupted ones) get served the just-written static `index.html` (queries ignored), so a small app converges instead of hanging. The loop needs corrupted requests to keep reaching SSR before static files shadow them.
- **Related output artifact:** all query-variant pages strip the query when computing the output path and overwrite the same `index.html`, while being reported as separately prerendered.

Posted the repro + analysis: [TanStack/router#7837](https://github.com/TanStack/router/issues/7837#issuecomment-5024448817). Offered a PR with regression test.
